redis内存管理-zmalloc

Posted on 2020-11-13 Edited on 2020-11-22 In redis Views:

redis在Linux平台上使用jemalloc作为默认的内存管理器，用户也可以自己修改使用tcmalloc、tcmalloc_minimal或者libc。

不管使用哪种内存管理器，redis都会进行统一的封装，使其对外的api保持一致。tcmalloc和jemalloc与c语言标准的内存分配api不一致，redis首先通过宏定义的方式统一映射，不同库的具体的映射关系如下：

zmalloc接口 tcmalloc接口 jemalloc接口 ptmalloc（libc） allocation zones接口
malloc(size) tc_malloc(size) je_malloc(size) ~ ~
calloc(count, size) tc_calloc(count, size) je_calloc(size) ~ ~
realloc(ptr, size) tc_realloc(ptr, size) je_realloc(size) ~ ~
free(ptr) tc_free(ptr) je_free(ptr) ~ ~
zmalloc_size(p) tc_malloc_size(p) je_malloc_usable_size(p) malloc_usable_size(p) malloc_size(p)

zmalloc_size(p)

zmalloc_size(p)函数返回指针p通过以上内存分配函数分配到的内存大小。随后redis对malloc、calloc、realloc和free进行二次封装，主要是在调用这些函数时，对分配/释放的总内存进行记录。tcmalloc, jemalloc,libc和mac os实现的库都有类似的函数，具体对应上图。但是如果上述实现都不可用时，redis使用自己定义的zmalloc_size(p)函数：

redis使用宏HAVE_MALLOC_SIZE标识这个函数是自定义的还是库自带的；如果定义了HAVA_MALLOC_SIZE，那么PREFIX_SIZE为0；如果没有定义，PREFIX_SIZE的值为sizeof(size_t)则分配内存时需要多分配PREFIX_SIZE的空间保存分配空间的大小。

redis中的内存记录

redis使用一个全局变量used_memory记录已经分配的总内存，由于可能会有多个线程同时访问这个变量，因此redis对used_memory读写操作保证原子性。

redis使用三种方式实现原子性，优先级依次递减（如果可用，优先使用__atomic宏实现，其次__sync宏，最后pthread mutex互斥锁），实现的具体方法在头文件atomicvar.h中，不同实现对应的操作如下：

redis对这三种方式进行了封装，对外提供了统一的api：

redis api __atomic __sync pthread mutex
atomicIncr(var,count) __atomic_add_fetch(&var, (count),__ATOMIC_RELAXED) __sync_add_and_fetch(&var,(count)) pthread_mutex_lock(&var ## _mutex);
var += (count);
pthread_mutex_unlock(&var ## _mutex);
atomicGetIncr(var, oldvalue_var, count) oldvalue_var=__atomic_add_fetch(&var, (count),__ATOMIC_RELAXED); oldvalue_var = __sync_fetch_and_add(&var,(count); pthread_mutex_lock(&var ## _mutex);
oldvalue_var = var;
var += (count);
pthread_mutex_unlock(&var ## _mutex);
atomicDecr(var, count) __atomic_sub_fetch(&var,(count),__ATOMIC_RELAXED) __sync_sub_and_fetch(&var,(count)) pthread_mutex_lock(&var ## _mutex);
var -= (count);
pthread_mutex_unlock(&var ## _mutex);
atomicGet(var, dstvar) dstvar=__atomic_load_n(&var, __ATOMIC_RELAXED); dstvar = __sync_sub_and_fetch(&var,0); pthread_mutex_lock(&var ## _mutex);
dstvar = var;
pthread_mutex_unlock(&var ## _mutex);
atomicSet(var,value) __atomic_store_n(&var, value, __ATOMIC_RELAXED) while(!__sync_bool_compare_and_swap(&var,var,value)); pthread_mutex_lock(&var ## _mutex);
var = value;
pthread_mutex_unlock(&var ## _mutex);

OOM错误处理

redis定义一个函数指针用来处理，该函数指针默认值为：

zmalloc(size_t size)

zcalloc(size_t size)

redis定义的zcalloc只有一个参数，与zmalloc(size_t size)较为类似，但是该函数会分配的空间初始化。

zrealloc(void *ptr, size_t size)

zfree(void *ptr)

zstrdup(const char *s)

redis定义的字符串复制函数，注意多分配了一个'\0'空间。
