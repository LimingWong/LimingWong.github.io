redis对象-string对象

Posted on 2020-12-23 Edited on 2021-03-11 Views:

redis中的字符串对象底层编码未必是sds字符串，这是因为redis为了节省内存，为不同的内容采用不同的编码。总的来说，字符串对象可能有三种不同的编码：

  • OBJ_ENCODING_RAW
  • OBJ_ENCODING_EMBSTR
  • OBJ_ENCODING_INT

redis首先会尝试编码为整数，如果可以的情况下优先使用共享对象。redis中关于整数有10000个共享对象，范围是0-9999. 然后会尝试使用embstr编码，字符串的长度小于等于44就可以编码为embstr。最后如果无法压缩这个字符串对象，那么尝试去掉字符串后面的空间，使用raw编码。
